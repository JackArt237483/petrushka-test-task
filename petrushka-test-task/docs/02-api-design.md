# 02. API Design — экран магазинов-партнеров

## 1. Цель

В рамках задания требуется спроектировать пример REST API метода, который вызывается при открытии экрана магазинов-партнеров в мобильном приложении интернет-магазина «Петрушка Зеленая».

На основании условия можно сделать вывод, что backend должен вернуть список магазинов-партнеров, пригодный для отображения на клиенте, включая ссылку для перехода на внешний ресурс при нажатии на карточку магазина.

---

## 2. Предположения

Так как в задании нет полного формального описания макета и полей, в рамках примера принимаются следующие допущения:

1. Экран отображает список магазинов-партнеров.
2. Для каждого магазина требуется:
    - идентификатор;
    - название;
    - краткое описание;
    - изображение / логотип;
    - ссылка на внешний ресурс.
3. При нажатии на карточку магазина мобильное приложение открывает внешний URL.
4. Метод только читает данные, поэтому для него используется `GET`.

---

## 3. Выбор REST API метода

Для получения списка магазинов-партнеров предлагается использовать следующий endpoint:

**HTTP Method:** `GET`  
**URL:** `/api/v1/partner-stores`

### Обоснование выбора

- используется метод `GET`, так как требуется только получение данных;
- путь `/partner-stores` явно отражает содержимое ресурса;
- версия API (`v1`) позволяет развивать контракт без ломки клиентов в будущем.
